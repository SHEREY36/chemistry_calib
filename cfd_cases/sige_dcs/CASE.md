## 3. Surface reaction mechanism

### 3a. Recommended: the UDF (`surface_bc_udf.c`)

This is the **exact** calibrated power law, transcribed to C, taking the
LOCAL wall temperature and LOCAL wafer-surface partial-pressure ratios CFD-
ACE+ computes each iteration (not the inlet setpoint) -- zero information
loss relative to the fitted model:

```c
double calibrated_GR_nm_min(double T_wall_K, double p_HCl_over_pDCS, double p_GeH4_over_pDCS) {
    const double lnK_GR     = 6.399306223;
    const double kappa_GR   = -1.494795145;   /* coefficient of STANDARDIZED 1/T */
    const double gamma_HCl  = -0.7097438995;
    const double gamma_GeH4 = 1.310385586;
    double invT_std = (1.0 / T_wall_K - 0.00104333196) / 6.105412345e-05;
    double ln_GR = lnK_GR + kappa_GR * invT_std
                 + gamma_HCl * log(p_HCl_over_pDCS) + gamma_GeH4 * log(p_GeH4_over_pDCS);
    return exp(ln_GR);   /* nm/min -- convert to your solver's flux units */
}
```
(`calibrated_xGe` alongside it, same file.) Register this as CFD-ACE+'s
surface boundary condition per your license's User Subroutines manual --
the exact registration mechanism is solver-version-specific and not
something I can specify without access to it; the arithmetic above is exact
regardless of that wrapper.

### 3b. Fallback: elementary mechanism (`elementary_mechanism.json`)

Use this only if your CFD-ACE+ setup needs a standard surface-reaction card
(species site fractions, individual step rates) rather than a UDF -- e.g.
for tracking adsorbed intermediates elsewhere in the model. **This form is
lossy**: a 4-parameter power law cannot uniquely determine 9 elementary rate
constants (an underdetermined inverse problem). Exactly ONE value here is
pinned by data; the rest are literature seeds to be independently verified
before trusting them quantitatively:

| Step | Reaction | Rate form | A (s^-1) | Ea (kcal/mol) | gamma (sticking) | Status |
|---|---|---|---|---|---|---|
| G1 | SiH2Cl2 <=> SiCl2 + H2 | Arrhenius (rev.) | -- | 65.0 | -- | seed |
| G2 | GeH4 <=> GeH2 + H2 | Arrhenius (rev.) | -- | 47.0 | -- | seed |
| S1 | SiH2Cl2(g) + 2s -> SiCl2(s) + 2H(s) | sticking | -- | -- | 0.03 | seed |
| S2 | SiCl2(s) -> Si(b) + 2Cl(s) | Arrhenius | 1e13 | 50.0 | -- | seed |
| **S3** | **2H(s) -> H2(g) + 2s** | **Arrhenius** | **1e13** | **48.65** | -- | **fit_to_power_law** |
| S4 | 2Cl(s) -> Cl2(g) + 2s | Arrhenius | 1e13 | 72.0 | -- | seed |
| S5 | HCl(g) + Si(b) -> SiHCl(s)/SiCl2(g) | Arrhenius | 1e12 | 30.0 | -- | seed |
| S6 | GeH4(g) + 2s -> Ge(s) + 2H(s) | sticking | -- | -- | 0.3 | seed |
| S7 | Ge(s) -> Ge(b) | Arrhenius | 1e12 | 15.0 | -- | seed |
| S8 | B2H6(g) + 2s -> 2BH3(s) -> 2B(b) | sticking | -- | -- | 0.1 | seed |
| S9 | H(s) + Cl(s) -> HCl(g) + 2s | Arrhenius | 1e13 | 58.0 | -- | seed |

**S3's Ea = 48.65 kcal/mol is derived, not guessed**: it equals
$-\kappa_{GR} \cdot R$ using the Phase 4 posterior mean
$\kappa_{GR}=-24{,}483\,\mathrm K$ (see METHODOLOGY.md sec 4 and
`chem_ml/cfd/mechanism.py:_calibrate_rate_limiting_step`), consistent with
the paper's own observation that H2 desorption is often the rate-limiting
step in this chemistry.

**Site conservation (enforce as a hard constraint):**
$$\theta_{\text{free}} + \theta_H + \theta_{Cl} + \theta_{Si} + \theta_{Ge} = 1$$

## 5. Feeding CFD-ACE+ output back into the ML pipeline

Once CFD-ACE+ has run, post-process its wafer-surface output into the
CSV contract `chem_ml/cfd/io.py` defines (`r_mm, surface_T_K,
p_HCl_over_pDCS, p_GeH4_over_pDCS, p_B2H6_over_pDCS, GR_nm_min, Ge_frac`),
then:
```bash
python -c "
from chem_ml.cfd.io import CFDCondition, parse_cfd_output
from chem_ml.cfd.transfer import extract_transfer_priors
cond = CFDCondition(T_set_K=973.15, flows_sccm={'DCS':50,'HCl':25,'GeH4':2.5,'B2H6':0}, P_tot_torr=10, geometry_id='AMAT_3D_v1')
result = parse_cfd_output('point3_wafer_profile.csv', cond)
priors = extract_transfer_priors([result])
print(priors)
"
```
Feed `priors` into `reactor_transfer_model_gr_ge(..., alpha_priors=priors)`
(or `add-reactor` once that CLI path accepts priors) in place of the weak
default -- this is what resolves the alpha/eta identifiability problem in
METHODOLOGY.md sec 8.
